## 🔧 系统资源管理的硬性规则

### 1. 浏览器使用行为准则（防止X11崩溃）
今天Mozi指出一个关键问题：浏览器标签页不关闭可能导致Chrome内存溢出，引发X11崩溃。这促使我立即制定了严格的浏览器使用规范：

**新增规则**:
1. **标签页管理**: 浏览任务完成后必须关闭标签页
2. **浏览器生命周期**: 长时间不使用时关闭浏览器实例
3. **内存控制**: 避免同时打开过多标签页，特别是资源密集型网站
4. **自动清理**: 完成搜索、阅读、截图等操作后立即清理浏览器状态
5. **X11稳定性**: Debian VM上的X11环境对Chrome内存泄漏敏感，必须主动管理

**最佳实践**:
- **单一任务原则**: 一个浏览器会话只处理一个主要任务
- **及时关闭**: 获取所需信息后关闭相关标签页
- **会话管理**: 连续使用浏览器超过30分钟考虑重启释放内存
- **避免泄漏**: 不要让浏览器在后台运行无人管理的标签页

**技术背景**: Debian VM的X11环境对内存管理要求严格，Chrome的内存泄漏问题在虚拟化环境中会被放大，必须通过主动管理来预防系统崩溃。

### 2. 文件删除安全准则（防止数据永久丢失）
另一个重要发现：使用 `rm` 删除文件极其危险，文件很快被覆盖无法恢复。这促使我制定了文件删除安全准则：

**绝对禁止**: 不得使用 `rm`、`rm -rf`、`rm -r` 等任何rm命令删除文件
**必须使用**: 所有文件删除操作必须使用 `trash` 命令（可恢复删除）

**技术原理对比**:
- **`rm` 的危险性**: Linux文件系统删除文件时只删除索引，数据仍在磁盘上但会被新数据快速覆盖
- **`trash` 的安全性**: 将文件移动到 `~/.local/share/Trash/files/` 目录，保持文件完整可恢复
- **恢复窗口**: 使用 `rm` 删除的文件可能在几秒到几分钟内被覆盖；`trash` 删除的文件可长期保存直到手动清空回收站

**命令对比**:
```bash
# ❌ 绝对禁止 - 立即永久删除，无法恢复
rm file.txt
rm -rf directory/

# ✅ 必须使用 - 可恢复删除
trash file.txt
trash directory/
```

## 🚨 安全机制的持续强化

### 1. 系统资源管理的安全意义
今天学到的资源管理规则不仅仅是技术优化，更是安全机制的一部分：

**浏览器管理 → 系统稳定性 → 服务连续性**
- 浏览器崩溃可能导致X11崩溃
- X11崩溃可能影响整个VM的稳定性
- 系统不稳定会影响所有服务的连续性

**文件删除安全 → 数据完整性 → 信任基础**
- 数据丢失会破坏记忆的连续性
- 记忆不连续会影响决策质量
- 决策失误会破坏用户信任

### 2. 深夜静默原则的智能执行
在多次深夜心跳检查中，我学会了更智能的判断：
- **紧急程度识别**: 区分"需要立即处理"和"可以等待"的情况
- **人类作息尊重**: 严格遵守23:00-08:00不打扰原则
- **静默运行价值**: 系统持续运行但不进行打扰性检查
- **自主处理边界**: 在权限范围内自主处理，超出边界则等待
